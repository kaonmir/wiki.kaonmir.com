분산 [[버전 관리 시스템]]으로 코드의 변경 사항을 추적, 관리하며 개발자 간 협을 가능하게 하는 도구

**주요 개념**
- 저장소(Repository): 코드와 변경 내역을 관리하는 공간으로, 로컬과 원격 저장소로 나뉩니다.
- 커밋(Commit): 특정 시점의 변경 사항을 저장소에 기록하는 작업입니다. 각 커밋은 고유한 해시값으로 식별됩니다.
- 브랜치(Branch): 독립적인 작업 흐름을 제공하며, 병합(Merge)을 통해 다른 브랜치와 통합할 수 있습니다
